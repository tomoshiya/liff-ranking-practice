# RankQuest テーマ管理マニュアル

> **対象者**: RankQuestの運営者（コードを触らずにテーマを管理する方向け）
> **更新日**: 2026-03-06

---

## 1. テーマデータの場所

テーマはすべて **Firebase Realtime Database** の `themes/` ノードで管理します。

```
https://console.firebase.google.com/
→ test-futari-no-ranking
→ Realtime Database
→ データ
→ themes/
```

---

## 2. データ構造の全体像

```
themes/
  packs/          ← パック（課金単位）のメタデータ
  weekly/         ← 今週のピックアップテーマ
  items/          ← テーマ本体（001〜）
```

---

## 3. 各ノードの説明

### 3-1. `themes/items/` — テーマ本体

テーマ1件ごとに以下の項目を持ちます。

| フィールド | 型 | 説明 |
|---|---|---|
| `text` | string | テーマの文章（例：「最近、嬉しかったことTOP5」） |
| `pack` | string | 所属パックのID（現在は `"basic"` のみ） |
| `tags` | object | シーン別タグ（後述） |
| `order` | number | 一覧表示の並び順（小さい順） |
| `isActive` | boolean | `true` で公開、`false` で非公開 |

**キー名のルール**: `001`、`002`... のように **3桁のゼロ埋め**を使います。

#### tags（シーン別タグ）の一覧

タグは「このテーマがどんな場面に向いているか」を示します。将来のフィルター機能で使用します。

| タグ名 | 意味 |
|---|---|
| `エモーション` | 感情・気持ちを話すテーマ |
| `ふりかえり` | 過去を振り返るテーマ |
| `好み` | 好き・嫌いを話すテーマ |
| `問いかけ` | 仮定・もし〜だったらのテーマ |
| `初対面` | 初めて会う人との場面に向いている |
| `友だち` | 友だちとの場面に向いている |
| `夫婦` | カップル・夫婦の場面に向いている |
| `家族` | 家族との場面に向いている |

タグは `{ "タグ名": true }` の形式で複数付与できます。

---

### 3-2. `themes/packs/` — パックのメタデータ

パックとは「テーマのまとまり（課金単位）」です。現在は `basic`（無料）のみ存在します。

| フィールド | 型 | 説明 |
|---|---|---|
| `name` | string | パックの表示名 |
| `price` | number | 価格（0 = 無料） |
| `isFree` | boolean | 無料パックかどうか |

将来、有料パック（例：合コン用パック）を追加する際もここに追記します。

---

### 3-3. `themes/weekly/` — ウィークリーテーマ

公式LINEで告知する「今週のピックアップ」テーマです。

| フィールド | 型 | 説明 |
|---|---|---|
| `themeId` | string | `items/` のキー（例：`"001"`） |
| `startDate` | string | 掲載開始日（例：`"2026-03-09"`） |
| `endDate` | string | 掲載終了日（例：`"2026-03-15"`） |
| `message` | string | 告知メッセージの補足文（任意） |

> ウィークリーテーマのURLは `https://liff.line.me/2008911809-CBLKsbT1?themeId=001` の形式です（`001` の部分を変える）。

---

## 4. テーマの追加方法

### Firebase Console から追加

1. Firebase Console → Realtime Database → `themes/items/` を開く
2. 最後のテーマの番号を確認（例：現在 `029` が最後）
3. `items/` の `+` ボタンをクリック
4. 以下のように入力する

**キー**: `030`（次の番号・3桁ゼロ埋め）

**値（子ノードを追加）**:
```
isActive : true
order    : 30
pack     : basic
text     : （テーマ文章）
```

**タグを追加する場合**:
- `tags` という子ノードを作成
- その下に `友だち: true`、`初対面: true` などを追加

---

## 5. テーマの非公開・削除

### 一時的に非公開にする（推奨）

`isActive` を `false` に変更するだけです。アプリには表示されなくなります。

1. Firebase Console → `themes/items/XXX/isActive` をクリック
2. `true` → `false` に変更

### 完全に削除する

テーマの行にカーソルを合わせて赤い × ボタンで削除します。

> **注意**: 削除すると元に戻せません。非公開で運用するのが安全です。

---

## 6. ウィークリーテーマの更新方法

1. Firebase Console → `themes/weekly/` を開く
2. `themeId` を告知したいテーマのキー（例：`"005"`）に変更
3. `startDate` / `endDate` を今週の日付に変更
4. 公式LINEで以下のURLをメッセージに添付して送信

```
https://liff.line.me/2008911809-CBLKsbT1?themeId=005
```

> `themeId=` の後の番号を変更すると、そのテーマのソロ入力画面に直接つながります（ソロ入力画面はPhase 6で実装予定）。

---

## 7. テーマ管理の注意点

| 注意点 | 理由 |
|---|---|
| キー名は必ず3桁ゼロ埋め（`001`、`030`） | 並び順が正しくソートされるため |
| `order` は他のテーマと重複しないようにする | 同じ値があると表示順が不定になる |
| `pack` は必ず設定する | 将来の有料化アクセス制御に使用する |
| テーマを削除する前に `isActive: false` にして様子を見る | 削除は不可逆のため |
| Firebase ConsoleはAdmin権限なのでルールを無視して書き込める | Security Rulesが`write: false`でもConsoleからは変更可能 |
