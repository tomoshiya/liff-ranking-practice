## あ行

### API（エーピーアイ）
**正式名称**: Application Programming Interface

**意味**: アプリケーション同士が会話するための「ルール」や「窓口」のこと。

**例え**: レストランのメニュー表のようなもの。  
- お客さん（アプリ）がメニュー（API）を見て注文する  
- キッチン（サーバー）が料理を作って提供する  
- お客さんはキッチンの中を知らなくても料理が食べられる

**具体例**: 
- LINE Messaging API: LINEにメッセージを送るための窓口
- LIFF API: LINEの中でWebアプリを動かすための窓口

## か行

### GitHub（ギットハブ）
**意味**: プログラムのコードを保存・管理するクラウドサービス。

**例え**: プログラムコードの「Google Drive」のようなもの。  
- コードをバックアップできる  
- 変更履歴が自動で残る  
- チームで共同開発できる  
- 無料でホームページを公開できる（GitHub Pages）

---

### GitHub Pages（ギットハブ ページズ）
**意味**: GitHubが提供する無料のホームページ公開サービス。

**例え**: 無料のレンタルサーバーのようなもの。  
- HTMLファイルをアップロードするだけで公開できる  
- https://ユーザー名.github.io/リポジトリ名/ というURLで公開される  
- 完全無料で使える

**このプロジェクトでの使い方**:  
LIFFアプリ（Webページ）を公開するために使っています。

## さ行

### SDK（エスディーケー）
**正式名称**: Software Development Kit（ソフトウェア開発キット）

**意味**: プログラムを作るための「道具箱」のこと。よく使う機能があらかじめ用意されている。

**例え**: プラモデルの「組み立てキット」のようなもの。  
- 部品（関数）が揃っている  
- 説明書（ドキュメント）がついている  
- 自分でゼロから作らなくても、組み立てるだけで使える

**具体例**:
- **LIFF SDK**: LINEアプリの中でWebページを動かすための道具箱
- **Firebase SDK**: Firebaseの機能（データベース、認証など）を使うための道具箱

**今回の作業**: Firebase SDKをHTMLファイルに追加（道具箱を開ける）

---

### script タグ / `<script>`（スクリプトタグ）
**意味**: HTMLファイルの中に「JavaScriptのプログラム」を書くための場所を示すタグ。

**例え**: 料理のレシピ本の中の「調理手順」のページのようなもの。

**使い方**:
```html
<script>
  // ここにJavaScriptのコードを書く
  alert("こんにちは！");
</script>
```

または、外部のプログラムファイルを読み込む：
```html
<script src="https://example.com/program.js"></script>
```

**今回の作業**: Firebase SDKを読み込むために `<script>` タグを使います。

## た行

### データベース（Database / DB）
**意味**: データを整理して保存しておく「倉庫」のこと。

**例え**: 図書館のようなもの。  
- 本（データ）を整理して保管  
- 目録（検索機能）で素早く見つけられる  
- 必要な時に取り出して、返却（保存）できる

**このプロジェクトでの使い方**:  
- ユーザーが入力したランキング  
- ユーザー同士のペア情報  
- ゲームの結果  

これらをFirebase Realtime Databaseに保存します。

## な行

### npm（エヌピーエム）
**正式名称**: Node Package Manager（ノード・パッケージ・マネージャー）

**意味**: JavaScriptの「道具（ライブラリ）」をインターネットからダウンロード・管理するためのツール。

**例え**: スマホの「App Store」や「Google Play」のようなもの。  
- 必要なアプリ（ライブラリ）を検索  
- インストール  
- アップデート  
が簡単にできる

**使い方**:
```bash
npm install firebase
```
→ 「firebaseという道具をインストールしてください」という命令

**今回の作業**: 
今回は npm は使いません！  
代わりに `<script>` タグで直接Firebaseを読み込みます。  
（npmは本格的な開発で使う方法で、今回はもっとシンプルな方法を使います）

## は行

### Firebase（ファイアベース）
**意味**: Googleが提供する「アプリ開発のための便利なサービス詰め合わせ」。

**例え**: レストラン経営に必要なものが全部揃っている「開業支援サービス」のようなもの。  
- 冷蔵庫（データベース）  
- レジ（認証システム）  
- 宅配サービス（プッシュ通知）  
- 自動調理器（自動実行機能）  

すべてが揃っていて、すぐに始められる。

**主な機能**:
- **Realtime Database**: データをリアルタイムで保存・同期
- **Authentication**: ログイン機能
- **Cloud Functions**: プログラムを自動実行
- **Cloud Messaging**: プッシュ通知

**このプロジェクトでの使い方**:  
- ランキングデータを保存  
- ユーザー同士でデータを共有  
- 週次で自動的にお題を配信（将来）

## ら行

### Realtime Database（リアルタイム データベース）
**正式名称**: Firebase Realtime Database

**意味**: データが「リアルタイム」（即座に）同期されるデータベース。

**例え**: Google ドキュメントの共同編集機能のようなもの。  
- Aさんが文字を入力すると、即座にBさんの画面にも反映される  
- インターネットにつながっていれば、常に最新の状態が見られる

**このプロジェクトでの使い方**:  
- あなたがランキングを入力  
- パートナーのアプリに即座に反映  
- お互いのデータが常に同期される

**通常のデータベースとの違い**:
- **通常**: データを保存したら、自分で「読み込む」ボタンを押さないと更新されない  
- **Realtime**: 保存したら自動的に全員の画面が更新される

### ライブラリ（Library）
**意味**: よく使うプログラムの部品を集めた「部品集」のこと。

**例え**: 料理の「合わせ調味料」のようなもの。  
- 醤油、みりん、砂糖を毎回測らなくても  
- 「焼肉のタレ」を買えば、すぐに使える

**具体例**:
- jQuery: HTML操作が簡単にできるライブラリ  
- Firebase SDK: Firebaseの機能を使うためのライブラリ

---

### LIFF（リフ）
**正式名称**: LINE Front-end Framework（ライン フロントエンド フレームワーク）

**意味**: LINEアプリの中でWebページを表示するための仕組み。

**例え**: LINEアプリの中に「ミニブラウザ」が入っているようなもの。  
- 普通のWebページを作る  
- LINEアプリで開くと、LINE内でページが表示される  
- LINE の機能（プロフィール取得、メッセージ送信など）が使える

**このプロジェクトでの使い方**:  
「RankQuest」はLIFFアプリです。  
LINEのトーク画面からリンクをタップすると、LINE内でゲームが開きます。

### LocalStorage（ローカルストレージ）
**意味**: ブラウザの中にデータを保存する「小さな引き出し」のこと。

**例え**: スマホの「メモ帳アプリ」のようなもの。  
- 自分のスマホだけに保存される  
- 他の人とは共有できない  
- アプリを閉じても消えない

**現在の使い方**:  
フェーズ2では、ランキングデータをLocalStorageに保存しています。

**今後の変更**:  
フェーズ3で、LocalStorage → Firebase Realtime Database に移行します。  
これにより、パートナーとデータを共有できるようになります。

**例え**: スマホアプリとの違い  
- **スマホアプリ**: App Storeからダウンロードして使う（Instagram、ゲームなど）  
- **Webアプリ**: ブラウザで開くだけで使える（Google マップ、YouTube など）

「RankQuest」はWebアプリです。

## アルファベット順

### HTML（エイチティーエムエル）
**正式名称**: HyperText Markup Language

**意味**: Webページの「骨組み」を作る言語。

**例え**: 家を建てる時の「設計図」のようなもの。  
- どこに見出しを置くか  
- どこにボタンを置くか  
- どこに画像を置くか  

を決める。

**具体例**:
```html
<h1>タイトル</h1>
<p>これは本文です。</p>
<button>クリックしてね</button>
```

---

### CSS（シーエスエス）
**正式名称**: Cascading Style Sheets

**意味**: Webページの「見た目（デザイン）」を決める言語。

**例え**: 家を建てた後の「内装・外装」のようなもの。  
- 壁の色  
- 家具の配置  
- 照明の明るさ  

を決める。

**具体例**:
```css
h1 {
  color: blue;        /* 文字色を青にする */
  font-size: 24px;    /* 文字サイズを24pxにする */
}
```

---

### JavaScript（ジャバスクリプト / JS）
**意味**: Webページに「動き」や「機能」をつける言語。

**例え**: 家の中の「電化製品」のようなもの。  
- ボタンを押したら電気がつく  
- スイッチを入れたらエアコンが動く  

といった「動作」を実現する。

**具体例**:
```javascript
// ボタンをクリックしたら「こんにちは！」と表示
button.onclick = function() {
  alert("こんにちは！");
};
```

---

### JSON（ジェイソン）
**正式名称**: JavaScript Object Notation

**意味**: データを「人間が読みやすく」「プログラムも扱いやすい」形式で表現する方法。

**例え**: Excel の「表」のようなもの。  
見やすく整理されていて、機械も読み取れる。

**具体例**:
```json
{
  "name": "太郎",
  "age": 25,
  "hobbies": ["サッカー", "読書", "料理"]
}
```

**このプロジェクトでの使い方**:  
- ランキングデータをJSON形式で保存  
- Firebaseとのデータのやり取りもJSON形式

---

### URL（ユーアールエル）
**正式名称**: Uniform Resource Locator

**意味**: インターネット上の「住所」のこと。

**例え**: 家の「郵便番号 + 住所」のようなもの。  
`https://tomoshiya.github.io/liff-ranking-practice/`  
この住所にアクセスすると、あなたのアプリが開く。
